# NextClaw 路线图

本文档描述 NextClaw 的产品方向与当前优先级。迭代历史见：[docs/logs](https://github.com/Peiiii/nextclaw/blob/master/docs/logs/README.md)。

---

## 当前优先级（按顺序）

### 1. 全量国际化 / 多语言能力

- UI、CLI、文档的国际化覆盖（如英语、中文）。
- 语言切换与持久化；文案、时间、数字格式一致化。

### 2. Landing 页面持续优化

- 更清晰的产品价值表达。
- 精选关键入口：快速开始、文档、GitHub、配置指南、示例场景，降低上手成本。

### 3. 内置聊天（核心产品能力）

- UI 支持直接与 AI 对话；配置 UI 继续保留，聊天成为与渠道、CLI 并列的一等入口。
- 支持“先聊后接入”，降低试用门槛。

### 4. 自知与自治

- **自知**：Agent 能查询自身版本、配置、运行状态、健康度、能力与文档入口。
- **自治**：Agent 可通过工具执行自身管理动作（改配置、启停渠道、增删 cron、诊断/重启等），对高风险操作需明确确认。

### 5. UI 与体验

- 持续优化配置 UI：布局、交互、反馈。
- 完善 Sessions、Routing & Runtime、Cron、Marketplace 信息架构与流程。
- 统一加载、保存、错误态体验。

### 6. 多 Agent 与多实例

- 多 Agent：会话隔离、bindings、路由与运行时语义持续增强并可验证。
- 多实例：部署模型、配置边界与最佳实践沉淀。

### 7. 生态：Skills & 插件 + 快速上手

- OpenClaw 兼容插件/skill 流程；支持在 UI 中安装、启用、配置。
- 完整“从安装到首个可用场景”的上手路径。

### 8. 文档：教程与示例

- 覆盖安装、配置、首个 bot、首个 cron 等步骤。
- 结合 Landing 与应用内帮助形成闭环路径：介绍 → 教程 → 示例。
