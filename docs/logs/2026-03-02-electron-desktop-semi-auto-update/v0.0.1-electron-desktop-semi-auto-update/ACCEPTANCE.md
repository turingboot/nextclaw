# 用户 / 产品视角的验收步骤

## 验收目标

- 确认方案覆盖“安装门槛低 + 升级可持续 + 单代码维护”的核心诉求。

## 步骤

1. 阅读方案文档：
   - [`docs/designs/electron-desktop-semi-auto-update-plan.md`](../../../designs/electron-desktop-semi-auto-update-plan.md)
2. 确认技术决策：
   - 桌面端选择 Electron。
   - 普通用户安装不依赖预装 Node/npm。
   - 更新策略为半自动更新，包含手动兜底。
3. 确认可执行边界：
   - 方案中明确了架构边界、发布策略、安全基线、阶段计划和验收标准。
4. 确认后续可落地性：
   - 文档给出分阶段目标，可直接作为实现任务拆分输入。

## 通过标准

- 产品、技术、交付三方对路线无关键分歧。
- 可基于该方案直接进入实现阶段，而无需再次进行方向性决策。
