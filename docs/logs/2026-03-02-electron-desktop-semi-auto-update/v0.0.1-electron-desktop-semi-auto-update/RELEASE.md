# 发布 / 部署方式

## 适用范围

- 本次为方案文档沉淀，不包含可执行代码发布。

## 执行动作

1. 将文档变更合并到主分支。
2. 在下一次桌面端实现迭代时，按本方案补齐构建脚本、打包配置和更新通道配置。

## 发布闭环判定

- NPM 发布：不适用（无包版本变更）。
- 安装器发布：不适用（无安装器构建变更）。
- 远程 migration：不适用（无后端/数据库变更）。
- 线上 API 冒烟：不适用（无服务端行为变更）。

## 后续发布前置条件（实现阶段）

1. 完成 Electron 壳层工程落地。
2. 完成无 Node 预装场景安装冒烟。
3. 完成半自动更新链路冒烟（检测、下载、重启安装、失败回退）。
