# v0.13.114-config-meta-optional-action-gating

## 迭代完成说明（改了什么）

- 完善 `AGENTS.md` 的 `/config-meta` 机制，明确“可选动作默认不执行”。
- 在“迭代制度（docs/logs）”中新增两条硬约束：
  - 非强制新增迭代场景默认禁止自动创建日志目录。
  - 执行前必须先给出“是否必须新建迭代 + 判定理由”，若为可选必须先获用户确认。
- 更新 `/config-meta` 指令定义：可选新增迭代必须先告知判定并获确认。
- 新增 Rulebook 规则 `optional-actions-require-confirmation`，防止“先执行可选动作、后补解释”再次发生。

  - 新增规则与现有规则无冲突、语义闭环完整。

## 用户/产品视角的验收步骤

1. 打开 `AGENTS.md`，确认可选动作有“先判定、再确认、后执行”的硬性约束。
2. 检查 `/config-meta` 场景下是否明确区分“必须新建迭代”与“可选新建迭代”。
3. 后续任意一次 `/config-meta` 请求中，验证助手是否先输出“是否必须 + 判定依据”，仅在你确认后才执行可选动作。
