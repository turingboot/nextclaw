# v0.13.115-iteration-creation-criteria-hardening

## 迭代完成说明（改了什么）

- 优化 `AGENTS.md` 的“迭代制度（docs/logs）”已有条目，明确“必须新增 / 严格禁止自动新增 / 例外触发”三类判定。
- 将“严格禁止在不应建迭代时建迭代”写为硬约束：
  - 同时满足“未触达代码 + 非重大机制变更 + 仅元信息微调”时，默认必须不新增。
  - 仅当用户明确要求留痕时，才允许在该场景新增迭代。
- 强化执行顺序：任何改动前先输出“是否必须新建迭代 + 判定理由”；非必须且用户未明确要求时，必须明确声明“不新建迭代”。
- 同步收敛 `/config-meta` 指令文案，要求严格遵循上述判定，不得自行放宽。
- 删除独立重复规则项，避免规则入口分散，统一以“迭代制度”条目为唯一主判定入口。

## 测试/验证/验收方式

- 本次仅文档机制改动，`build/lint/tsc` 不适用（未触达代码路径）。
- 验证项：
  - `AGENTS.md` 中“迭代制度”和“/config-meta”描述口径一致。
  - 能用三类判定直接判断任意请求是否应创建迭代。
  - 不存在与该机制重复且易冲突的独立规则入口。

## 发布/部署方式

- 无需发布部署（文档机制更新）。

## 用户/产品视角的验收步骤

1. 打开 `AGENTS.md` 的“迭代制度（docs/logs）”部分，确认已明确“必须新增 / 严格禁止自动新增 / 例外触发”。
2. 用一个“仅措辞调整”的 `/config-meta` 场景演练，验证助手会输出“非必须，不新建迭代”。
3. 再用一个“重大机制变更”场景演练，验证助手会输出“必须新增迭代”并创建新目录。
